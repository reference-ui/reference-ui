# Strict Token Categories

## Background

Panda CSS exposes a single `strictTokens` boolean that blocks all arbitrary values across every token category. That is too coarse — a team may want to enforce design tokens for brand-sensitive props (colors, radii) while still allowing freeform values for utility props (width, margin, etc.).

Reference UI already solves this for colors. `StrictColorProps<P>` is a TypeScript utility that replaces every color-bearing style prop's value union with `StrictColorValue` — restricting it to design tokens plus a small set of semantic CSS keywords (`white`, `black`, `inherit`, `currentColor`, `transparent`). This is generated at `ref sync` time and composed into `SystemStyleObject`.

The goal here is to make that mechanism configurable and extend it to `radii` and optionally `spacing`.

---

## Proposed API

```ts
// ui.config.ts
export default defineConfig({
  name: 'my-app',
  include: ['src/**/*.{ts,tsx}'],
  strict: ['colors', 'radii'],
})
```

`strict` is an optional array of token category names. Each entry narrows the style props that draw from that category to token values only (plus a small set of safe escape hatches per category). Omitting `strict` leaves all props open, matching current behaviour.

---

## Supported categories

### `colors` (already implemented)

Style props: all `ColorPropKeys` (`color`, `bg`, `backgroundColor`, `borderColor`, `fill`, `stroke`, etc.)

Escape hatches: `white`, `black`, `inherit`, `currentColor`, `transparent`

Implementation: `StrictColorProps<P>` in `packages/reference-core/src/types/colors.ts`, applied via `SystemStyleObject`.

---

### `radii`

Style props: `borderRadius`, `borderTopLeftRadius`, `borderTopRightRadius`, `borderBottomRightRadius`, `borderBottomLeftRadius`, `rounded`, `roundedTop`, `roundedRight`, `roundedBottom`, `roundedLeft`, `roundedTopLeft`, `roundedTopRight`, `roundedBottomRight`, `roundedBottomLeft`

Escape hatches: `none`, `inherit`, `initial`, `revert`

Rationale: Radii define the overall shape vocabulary of the palette. Locking them to tokens prevents ad-hoc pixel values that diverge from the design language, in the same way strict colors prevent off-palette hex strings.

---

### `spacing` (optional / lower priority)

Style props: `margin`, `m`, `mx`, `my`, `mt`, `mr`, `mb`, `ml`, `padding`, `p`, `px`, `py`, `pt`, `pr`, `pb`, `pl`, `gap`, `rowGap`, `columnGap`

Escape hatches: `0`, `auto`, `inherit`, `initial`, `revert`

Rationale: Spacing is worth considering for design systems that want a locked rhythm scale, but it's higher friction than colors/radii because spacing is used very broadly. Mark as opt-in and low priority.

---

## Implementation sketch

Each entry in `strict` maps to a companion type utility analogous to `StrictColorProps`:

```ts
// generated by ref sync into .reference-ui/types/strict.d.ts
export type StrictTokenProps<P> =
  & StrictColorProps<P>        // when 'colors' in strict
  & StrictRadiiProps<P>        // when 'radii' in strict
  // & StrictSpacingProps<P>   // when 'spacing' in strict
```

`StrictRadiiProps<P>` follows the same pattern as `StrictColorProps<P>`:

```ts
type RadiiPropKeys = 'borderRadius' | 'borderTopLeftRadius' | /* … */ | 'rounded' | 'roundedTop' | /* … */

type RadiusToken = UtilityValues['borderRadius']

type StrictRadiusValue =
  | RadiusToken
  | 'none'
  | 'inherit'
  | 'initial'
  | 'revert'

export type SafeRadiiProps = {
  [K in Extract<keyof SystemProperties, RadiiPropKeys>]?: StylePropValue<StrictRadiusValue>
}

export type StrictRadiiProps<P> = Omit<P, RadiiPropKeys> & SafeRadiiProps
```

`SystemStyleObject` then becomes:

```ts
export type SystemStyleObject = StrictTokenProps<StyledSystemStyleObject>
```

The `strict` array from `ui.config.ts` is read during `ref sync`; the codegen step emits the composed `StrictTokenProps` type with only the requested categories active.

---

## Open questions

- Should `strict: []` be valid (explicitly opting out of everything, including the current color default)? Probably yes — gives teams an escape hatch if they need it.
- Does `radii` strictness also apply to `css()` and `cva()`? Yes — `SystemStyleObject` is the shared base, so it applies everywhere that type flows.
- Spacing: defer until there is a concrete team request. Colors + radii cover the palette identity.
