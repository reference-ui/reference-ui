# Icons

This page smoke-tests the Material icon components shipped in `@reference-ui/icons`, re-exported for docs through **`@reference-ui/lib/icons`** so the docs app does not depend on the icons package directly.

<P fontSize="sm" color="docsMuted" marginTop="0" marginBottom="4r">
  `reference-docs` depends only on `@reference-ui/lib`. Icons resolve transitively from the library’s
  dependency on `@reference-ui/icons`, and the docs build runs after `reference-lib` builds (which
  includes a full `reference-icons` build so packaged entrypoints exist).
</P>

## Tokens, background, and `size`

Material icons render with `fill="currentColor"`, so **`color` on a parent** (for example `green.500`) controls the glyph. **`size`** maps to **`width` and `height`** on the icon’s outer **`Div`**, so tokens (`5r`), **`rem`**, numbers, **`calc()`**, etc. behave like everywhere else. The design system extended by **`@reference-ui/lib`** already layers in the icons **`baseSystem`**, so **`ref sync`** in docs picks up the same styling behavior without listing `reference-icons` sources in this package’s **`include`**.
