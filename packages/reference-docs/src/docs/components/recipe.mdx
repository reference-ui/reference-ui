# Recipe

Build multi-variant styles with `cva()` (or `recipe()`). Define base styles, variants, compound variants, and default variants; use the returned function as `className={buttonRecipe({ visual: 'solid', size: 'md' })}`.

## Live demo

<RecipeDemo />

## Basic usage

```tsx
import { cva } from '@reference-ui/react'

const buttonRecipe = cva({
  base: {
    px: '4r',
    py: '2r',
    borderRadius: 'md',
    cursor: 'pointer',
  },
  variants: {
    visual: {
      solid: { bg: 'blue.500', color: 'white' },
      outline: { bg: 'transparent', borderColor: 'blue.500' },
    },
    size: {
      sm: { px: '3r', py: '1.5r' },
      md: { px: '4r', py: '2r' },
      lg: { px: '6r', py: '3r' },
    },
  },
  defaultVariants: {
    visual: 'solid',
    size: 'md',
  },
})

// Use in your component
<button className={buttonRecipe({ visual: 'outline', size: 'lg' })}>
  Click me
</button>
```

## recipe() vs cva()

You can use Panda's `cva` directly from `@reference-ui/react` (or `@reference-ui/react/css`). The `recipe()` export is a thin wrapper that keeps the same API and can be used when you want a single entrypoint; both work with the design tokens and extraction pipeline.

## Compound variants

Apply styles when multiple variants are combined:

```tsx
const myRecipe = cva({
  variants: {
    visual: { solid: { bg: 'blue.500' }, outline: { borderColor: 'blue.500' } },
    colorScheme: { red: {}, blue: {} },
  },
  compoundVariants: [
    {
      visual: 'outline',
      colorScheme: 'red',
      css: { borderColor: 'red.500', color: 'red.600' },
    },
  ],
})
```
