# CSS

The `css()` function is your primary styling primitive in Reference UI. It takes a style object and returns a class name string that you can apply to any element.

## Live demo

<CssDemo />

## Basic usage

```tsx
import { css } from '@reference-ui/react'

const styles = css({
  padding: '4r',
  bg: 'blue.500',
  color: 'white',
  borderRadius: 'md',
  fontWeight: '600',
})

// Use in your component
<div className={styles}>
  Styled element
</div>
```

## Design tokens

Reference UI provides a comprehensive set of design tokens that work seamlessly with `css()`:

### Colors

```tsx
css({
  bg: 'blue.500', // Background color
  color: 'white', // Text color
  borderColor: 'gray.300', // Border color
})
```

Available color scales: `slate`, `gray`, `zinc`, `neutral`, `stone`, `red`, `orange`, `amber`, `yellow`, `lime`, `green`, `emerald`, `teal`, `cyan`, `sky`, `blue`, `indigo`, `violet`, `purple`, `fuchsia`, `pink`, `rose`

Each scale has shades: `50`, `100`, `200`, `300`, `400`, `500`, `600`, `700`, `800`, `900`, `950`

### Spacing

Use the rhythm-based spacing system with the `r` suffix:

```tsx
css({
  padding: '4r', // 4 * rhythm units
  margin: '2r', // 2 * rhythm units
  gap: '3r', // 3 * rhythm units
})
```

Or use Panda's numeric spacing:

```tsx
css({
  padding: '4', // 1rem (16px)
  margin: '2', // 0.5rem (8px)
})
```

### Typography

```tsx
css({
  fontSize: 'xl', // Font size tokens: xs, sm, md, lg, xl, 2xl, 3xl, 4xl...
  fontWeight: '600', // Font weights: 100-900
  lineHeight: '1.5', // Line height
  letterSpacing: '-0.02em', // Letter spacing
})
```

## Responsive styles

Use an object with breakpoint keys to create responsive styles:

```tsx
css({
  padding: { base: '2r', md: '4r', lg: '6r' },
  fontSize: { base: 'md', md: 'lg', lg: 'xl' },
  display: { base: 'block', md: 'flex' },
})
```

Default breakpoints:

- `base`: 0px (mobile-first)
- `sm`: 640px
- `md`: 768px
- `lg`: 1024px
- `xl`: 1280px
- `2xl`: 1536px

## Pseudo states

Style hover, focus, active, and other states with the `_` prefix:

```tsx
css({
  padding: '3r 6r',
  bg: 'blue.600',
  color: 'white',
  transition: 'all 0.2s',
  _hover: {
    bg: 'blue.700',
    transform: 'translateY(-2px)',
  },
  _focus: {
    outline: '2px solid',
    outlineColor: 'blue.500',
  },
  _active: {
    transform: 'translateY(0)',
  },
})
```

Available pseudo states:

- `_hover`, `_focus`, `_active`, `_disabled`
- `_focusVisible`, `_focusWithin`
- `_before`, `_after`
- `_first`, `_last`, `_even`, `_odd`
- `_visited`, `_checked`, `_indeterminate`
- And many more...

## Layout

### Flexbox

```tsx
css({
  display: 'flex',
  flexDirection: 'row',
  justifyContent: 'center',
  alignItems: 'center',
  gap: '4r',
})
```

### Grid

```tsx
css({
  display: 'grid',
  gridTemplateColumns: 'repeat(3, 1fr)',
  gap: '4r',
})
```

### Positioning

```tsx
css({
  position: 'relative',
  top: '0',
  left: '0',
  zIndex: '10',
})
```

## Advanced features

### Style composition

You can compose styles by calling `css()` multiple times or using arrays:

```tsx
const baseStyles = css({
  padding: '4r',
  borderRadius: 'md',
})

const variantStyles = css({
  bg: 'blue.500',
  color: 'white',
})

// Apply both
<div className={`${baseStyles} ${variantStyles}`}>
  Composed styles
</div>
```

### Conditional styles

```tsx
const [isActive, setIsActive] = useState(false)

<div className={css({
  bg: isActive ? 'green.500' : 'gray.200',
  color: isActive ? 'white' : 'gray.700',
})}>
  Conditional styling
</div>
```

### Raw styles

Access the style object without generating a class name:

```tsx
const styleObject = css.raw({
  padding: '4r',
  bg: 'blue.500',
})

console.log(styleObject) // { padding: '1rem', backgroundColor: 'var(--colors-blue-500)' }
```

## Best practices

1. **Use design tokens**: Prefer token values (`'blue.500'`) over hard-coded colors
2. **Mobile-first**: Start with base styles and add larger breakpoints
3. **Semantic naming**: Use meaningful variable names for your styles
4. **Compose wisely**: Break complex styles into smaller, reusable pieces
5. **Leverage pseudo states**: Use `_hover`, `_focus`, etc. for interactive elements

## css() vs JSX props

For simple cases, you can use JSX props directly on Reference UI components:

```tsx
// Using JSX props (simpler for one-off styles)
<Div padding="4r" bg="blue.500" color="white">
  Content
</Div>

// Using css() (better for reusable styles or complex compositions)
<div className={css({ padding: '4r', bg: 'blue.500', color: 'white' })}>
  Content
</div>
```

Both approaches work with the same design tokens and responsive syntax!
