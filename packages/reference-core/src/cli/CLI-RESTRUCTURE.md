# CLI RESTRUCTURE

**Goal:** Replace monkey-patching and workarounds with a clean architecture where **primitives** and **styled** depend on **system** (Panda-generated). The CLI owns the eval loop, config generation, and bundling.

---

## The Core API

The core API lives in:

- **`styled/`** — tokens, recipes, patterns, font, keyframes, globalCss, utilities, rhythm, theme
- **`primitives/`** — typographic recipes (h1–h6, p, strong, code, etc.)

Both depend on **system**, which is generated by Panda. System includes:

- `system/css` — `css`, `cva`, `cx`, `sva`
- `system/jsx` — Box, Stack, styled factory
- `system/patterns` — box pattern
- `system/recipes` — compiled recipe output
- `system/types` — `SystemStyleObject`, `RecipeVariantRecord`, etc.
- `system/tokens`
- `system/helpers`

System is **Panda output** (`outdir: 'src/system'`); it doesn't exist as source.

---

## The Chicken-Egg Problem

1. **Panda generates system** from config.
2. **Config loading** (eval) runs `panda.base.ts` → `styled/index` → theme, font, primitives, etc.
3. **`styled/index`** pulls in:
   - `styled/theme/*` → `extendTokens`, etc.
   - `styled/css.global` → `import { extendGlobalCss } from './api'`
   - `styled/api` → `export * from './runtime'` → `css.ts`, `recipe.ts`
4. **`styled/api/runtime/css.ts`** imports `../../../system/css/index.js`.
5. **System doesn't exist yet** when eval runs → import fails.

So: system is produced by Panda, but loading the config that Panda needs pulls in code that imports system.

Same for **primitives**: `primitives/index.tsx` imports from `system/helpers`, `system/patterns/box`, `system/jsx`, `system/recipes`. The recipe definitions use `defineRecipe` (no system), but the primitives runtime entry needs system to render.

---

## Why Previous Isolation Attempts Failed

From [ISOLATION.md](ISOLATION.md) and related docs:

- Isolation focused on per-consumer output dirs and config paths.
- **primitives + styled as a coherent unit** were not factored in.
- Primitives live in a separate folder with the expectation that "system always exists."
- Styled does a lot and also needs system to exist within `src`.
- Attempts to isolate without a clean split between **build-time extension** vs **runtime consumption** led to circular dependencies and workarounds.

---

## Current Workarounds (What We Want to Move Away From)

1. **Bootstrap stubs** — Write minimal stubs for `system/css`, `system/types`, etc. so imports resolve before Panda runs.
2. **Packager-first** — Wait for packager to complete so `@reference-ui/system` exists before eval. Packager itself needs `src/system/` when bundling react (which imports from system).
3. **Ordering hacks** — Eval, createPandaConfig, gen, packager choreographed via events.

These work but are brittle and hard to reason about.

---

## Proposed Direction: CLI Owns the Eval Loop and System

### Principle

**The CLI is the integration point.** primitives and styled should flow through the **bundler/packager** and **system module** in the CLI—not assume system exists in source.

### 1. Split Config-Time vs Runtime Imports

Config-time code (eval) should **never** import runtime code that depends on system:

| Layer           | Config-time (eval)       | Runtime (browser)              |
|-----------------|--------------------------|--------------------------------|
| styled/theme    | extendTokens              | —                              |
| styled/api      | extendRecipe, extendFont…| css, recipe (need system)      |
| primitives      | recipes (defineRecipe)    | index.tsx (needs system)       |

Right now, `styled/css.global` imports from `./api`, which pulls in runtime. Config-time paths should import only from `api/internal/*` (extenders) and never from `api/runtime`.

### 2. CLI-Generated Base Config

`reference-core` generates a base config from what we extend in Panda:

- Scan `styled/` and `primitives/` for extenders.
- Run eval in a **config-only** context (no system needed).
- Merge fragments into base config.
- Panda runs on that → generates system.

### 3. Bundler/Packager as the Glue

- **primitives** and **styled** are source.
- **system** is generated output.
- The **packager** bundles them with correct system resolution:
  - Eval entry: config-only, no `system/*` imports.
  - React package: `react.ts` → primitives, system, styled runtime.
  - System package: build-time APIs (tokens, recipe, etc.) — no Panda system.

So: eval never needs system. Packager consumes system and produces the final packages. primitives + styled integration happens through the CLI pipeline, not by assuming "system always exists."

### 4. Clean Module Boundaries

```
┌─────────────────────────────────────────────────────────────────┐
│  CLI (eval + gen + packager)                                     │
│                                                                  │
│  1. Eval (config-only) → styled/theme, styled/api/internal,      │
│     primitives/recipes → extendPandaConfig fragments            │
│  2. createPandaConfig → merged config                             │
│  3. Panda generate → src/system/                                 │
│  4. Packager → bundles react (primitives + styled runtime + system)│
└─────────────────────────────────────────────────────────────────┘
```

---

## Implementation Outline

1. **Split styled/api exports**
   - Config-only entry for eval: `api/internal/*` + extenders only.
   - Runtime entry: `api/runtime/*` (depends on system).
   - Update `styled/index`, `css.global`, etc. so eval never imports runtime.

2. **Config-only eval entry**
   - Dedicated entry for eval (e.g. `panda.config.entry.ts`) that imports only config extenders.
   - No `styled/api` barrel that re-exports runtime during eval.

3. **CLI system module**
   - CLI owns the "system" concept: when it exists, where it lives, when it's ready.
   - Packager resolves `system/*` via CLI-provided system path (core or per-consumer).

4. **primitives + styled as unified consumers**
   - Both treated as "consumers of system" that go through the packager.
   - Primitives recipes = config. Primitives components = runtime.
   - Same split as styled: config vs runtime.
